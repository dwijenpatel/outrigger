One design note worth flagging: the spec's ARG RESOLUTION rule states a non-matching string arg becomes a *literal* (not an error) — this is required for the worked example's `concat` separator `" "` to resolve correctly. Under that rule, a genuinely missing source field degrades to a literal (its own field-name string) rather than raising immediately; it only surfaces as a captured error once a type-sensitive op (divide, to_date, cast, etc.) rejects that literal. I implemented it exactly per the pinned rule and covered this behavior in `test_ops.py::MissingFieldTest` using `divide`.
